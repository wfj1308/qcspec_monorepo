# CoordOS / DocPeg 研发执行拆解（Web + Mobile）

## 1. 目标
把《产品需求总纲》转成可执行研发计划，覆盖后端、Web、移动端、数据、测试与运维，支持 `P0 -> P1 -> P2` 连续交付。

## 2. 团队分工

| 角色 | 责任域 | 交付物 |
|---|---|---|
| 后端组 | Gate、SignPeg、ExecutorPeg、Conservation、离线合拢 | API、规则引擎、任务调度、台账 |
| Web组 | SpecIR Studio、全息工作台、台账审计、归档中心 | 管理端页面、可视化、权限面板 |
| 移动组 | SDUI 渲染、现场采集、离线队列、签名提交 | App 页面容器、协议引擎、离线能力 |
| 数据/Infra组 | v://解引用、事件总线、告警、观测 | 表结构、索引、SLO 仪表盘 |
| QA组 | 端到端链路与角色回归 | 用例库、自动化脚本、验收报告 |

## 3. 里程碑计划

## P0（2-3 周）主权执行链路跑通
1. 后端
   - 统一角色鉴权中间件（接口强校验）
   - SignPeg 签名/验签/状态接口闭环
   - Gate 规则统一执行入口（前后端同源）
   - 施工资料/监理资料签署链模板下发（含 1-8 / 1-7 串行位）
   - 评定表来源位映射与自动回填能力（施工/监理）
   - 多参建组织主账号（Org Admin）与子账号自治 API（施工/监理/业主/检测/分包/设计）
2. Web
   - 角色分权菜单与按钮一致化
   - SignPeg 签名台账与状态可视化
   - 签字区按签署链位展示“已签/待签/阻断原因”
   - 组织账号管理台（成员创建、角色分配、项目绑定，按组织类型模板切换）
3. 移动
   - SDUI 协议渲染最小可用版
   - 工单 -> 填报 -> 拍照 -> 签名 -> 提交流程
   - 离线缓存与补传基础能力
   - 按签署链位渲染签名步骤与禁用原因提示
   - 登录后执行体选择（同账号多执行体场景）
4. QA
   - 样板链路：桥施7表单链路全通
   - 三角色回归：OWNER/SUPERVISOR/AI

验收：
1. 现场可不依赖纸质完成一条 Trip 链。
2. Web 可实时看到签名、Gate、Proof 状态。

## P1（3-5 周）守恒对账与自动拦截
1. 后端
   - IQC + 材料 UTXO 规则接入 Gate
   - Conservation Gate（偏差阈值 + 熔断）
   - ClawPeg 告警与仲裁池
2. Web
   - SpecIR Studio（Schema/QCGate/BOM）
   - Ref-Link 影子挂载工作台
3. 移动
   - SDUI 扩展：动态步骤、动态字段校验
   - 冲突回放与失败重试页
4. QA
   - 超耗、跳步、重复领料、冲突合并用例

验收：
1. 偏差超阈值自动阻断结算流转。
2. 材料去向可追溯到构件级。

## P2（4-8 周）商业闭环与监管穿透
1. 后端
   - FormulaPeg 计价规则引擎
   - RailPact 支付指令与争议剥离
   - DocFinal 归档编译与 CA 编排
2. Web
   - 结算台账、监管审计视图、Public Verify 管理
3. 移动
   - 国际化（中/英）与监管抽查入口
4. QA
   - 归档与验真全链路压力测试

验收：
1. 可输出可支付、可审计、可验真的完整归档包。

## 4. P0 任务清单（可直接开工）

## 4.1 后端
1. 为关键路由补齐 `required_role` 与 `required_scope` 元数据。
2. 新增统一鉴权装饰器，拒绝前端绕过。
3. 固化 SignPeg 错误码：
   - `skill_mismatch`
   - `capacity_full`
   - `cert_expired`
   - `role_not_allowed`
4. 建立 Trip 事件落表与幂等键。
5. 增加组织主账号与子账号管理接口（组织内自治）。

## 4.2 Web
1. 菜单分权和页面动作分权一一对齐（含 toast 与禁用态）。
2. SignPeg 面板显示：
   - 当前角色可签位
   - 已签状态
   - 拒绝原因
3. 增加权限可视化总览页（角色-页面-按钮矩阵）。

## 4.3 移动
1. 定义 SDUI 最小协议：
   - `steps[]`
   - `actions[]`
   - `field_rules[]`
   - `blocking_reasons[]`
2. 实现协议渲染器（不写死业务页面）。
3. 建立离线包格式（本地事务队列 + 重放状态）。
4. 增加 SDUI 协议版本协商与降级策略（版本不兼容时切换安全最小指令集）。

## 4.4 QA
1. 核心链路自动化：
   - 在线流程
   - 断网录入 + 联网补传
   - 权限越权回归
2. 验收脚本产出：
   - 角色脚本
   - 表单脚本
   - 签名脚本

## 5. 关键技术契约

1. 协议契约
   - 移动端只消费协议，不消费业务硬编码。
2. 事件契约
   - 所有状态变更必须有 Trip 事件与 Proof 关联。
3. 结算契约
   - 无 Conservation Gate 通过，不得触发 RailPact。
4. 归档契约
   - 归档节点必须 CA，不得用 SignPeg 代替。

## 6. Definition of Done（DoD）

1. 功能完成
   - 通过单元测试 + 集成测试 + E2E 回归。
2. 规则完成
   - 权限、Gate、签名、守恒规则均有负向用例。
3. 可观测完成
   - 接口日志、错误码、告警埋点齐全。
4. 文档完成
   - API 文档、协议样例、错误码字典同步更新。

## 7. 质量与SLO

1. P0 目标
   - Trip 提交成功率 >= 99.5%
   - 签名成功率 >= 99.0%
   - 离线补传成功率 >= 99.0%
2. P1 目标
   - 守恒拦截误报率 <= 2%
   - 告警到达率 >= 99.9%
3. P2 目标
   - Public Verify 可用性 >= 99.9%
   - 结算穿透到 Trip 证据链路查询成功率 >= 99.9%
   - 穿透链路响应时间：P95 <= 2s（同城）/ <= 4s（跨域），P99 <= 5s

## 8. 风险清单与应对

1. 风险：前后端权限不一致  
   - 应对：接口级强鉴权 + 前端只做展示分权。
2. 风险：离线冲突难复现  
   - 应对：引入可回放日志与冲突模拟器。
3. 风险：规则频繁变更  
   - 应对：SpecIR 版本化 + 灰度发布。
4. 风险：CA 集成时序复杂  
   - 应对：先落签章编排状态机，再接具体 CA 厂商。

## 9. 本周建议动作

4. 每两天一次端到端冒烟回归，防止“边改边散”。
